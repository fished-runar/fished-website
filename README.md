# Fished Website

Marketing site for [Fished](https://fished.com) — cloud-based software for the seafood industry.
Built with [Eleventy](https://www.11ty.dev/) (static site generator) and [Tailwind CSS v4](https://tailwindcss.com/), deployed on Cloudflare Pages.

## Quick start

```powershell
npm install
npm run dev
```

`npm run dev` watches everything (templates, data, and CSS) and rebuilds automatically — this is the command you want while actively working on the site. Then open the URL Eleventy prints in the terminal (defaults to `http://localhost:8080`).

On Windows, double-clicking [run.bat](run.bat) does the same thing (installs dependencies on first run, opens the browser, starts the dev server).

## Project structure

```
fished-website/
├── *.njk                  Page templates — one per URL (index.njk → /, about.njk → /about.html, ...)
├── _includes/
│   ├── layouts/base.njk    Shared HTML shell used by every page (<head>, nav, footer, global JS)
│   └── components/*.njk    Reusable page fragments (nav, footer, cards, CTA strip, ...)
├── _data/*.json            Site content — team members, testimonials, events, news, nav links, etc.
├── styles/main.css         Tailwind SOURCE stylesheet (this is the file you edit)
├── assets/
│   ├── css/main.css        Tailwind COMPILED output (generated — do not edit directly)
│   ├── photos/, fished/, graphics/, customers/, integrations/   Images used across the site
├── functions/api/          Cloudflare Pages Functions (serverless endpoints, e.g. the contact form)
├── scripts/                One-off maintenance/build scripts
├── _site/                  Build output (generated by Eleventy — never edit, never commit)
├── eleventy.config.js       Eleventy configuration (passthrough copies, custom filters, output dir)
└── package.json             npm scripts and dependencies
```

## Editing content

Most day-to-day content changes don't require touching any template — just edit the relevant JSON file under `_data/`:

| Content | File |
|---|---|
| Events (trade shows, conferences) | `_data/events.json` |
| Testimonials | `_data/testimonials.json` |
| Team members | `_data/team.json` |
| News articles | `_data/news.json` |
| Office locations | `_data/offices.json` |
| Top nav / footer links | `_data/nav.json` |

Each file is a JSON array of objects — to add an entry, copy an existing object and edit its fields; to remove one, delete it. Look at an existing entry in the same file as your template for the expected schema (e.g. events use `title`, `date`, `displayDate`, `location`, `stand` (optional), `link`).

Images referenced from data files or templates live under `assets/` and are referenced with an absolute path, e.g. `/assets/photos/team.png`.

**No rebuild needed** for `_data/*.json` or `*.njk` template edits while `npm run dev` (or `npm run eleventy:serve`) is running — Eleventy picks up the change and live-reloads automatically.

## Editing styles

`styles/main.css` is the only stylesheet you should edit — it's the Tailwind source file. The browser actually loads the **compiled** output at `assets/css/main.css`, which Tailwind generates from the source.

- If `npm run dev` (or `npm run css:watch`) is running, this happens automatically on save.
- If you're only running `npm run eleventy:serve` on its own, you must manually recompile after editing CSS:
  ```powershell
  npm run css
  ```
  Otherwise the browser keeps showing stale styles with no error or warning.

## npm scripts

| Command | What it does |
|---|---|
| `npm run dev` | Watches CSS + templates + data together, auto-rebuilds on save (recommended for active editing) |
| `npm run eleventy:serve` | Starts the Eleventy dev server only, without CSS watching |
| `npm run css` | One-shot Tailwind compile (`styles/main.css` → `assets/css/main.css`) |
| `npm run css:watch` | Same as above, but recompiles on every save |
| `npm run build` | Production build: minified CSS + full Eleventy build into `_site/` |
| `npm run check:assets` | Runs `scripts/check-asset-paths.js` to catch broken/missing asset references |

## Deployment

The site is deployed on **Cloudflare Pages**. Pushing to `main` triggers a production deploy (build command: `npm run build`, output directory: `_site`); a `dev` branch exists for previews. `functions/api/*.js` files are deployed as Cloudflare Pages Functions (currently just the contact form proxy).

`_headers` configures caching and security headers for the deployed site (Cloudflare Pages reads this directly).

## Git workflow

```powershell
git status                                    # see what changed
git add -A
git commit -m "Describe what changed"
git push origin main                          # deploy

git restore <file>                            # discard uncommitted local edits
git revert <hash>; git push origin main       # safely undo an already-pushed commit
```

Avoid `git reset --hard` or `git push --force` on `main` — use `git revert` instead, since it doesn't rewrite shared history.
